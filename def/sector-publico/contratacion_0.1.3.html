Abstract

The CONTSEM ontology defines the necessary concepts to descrube public procurement contracct and various elements needed to describe its procedure.

The ontology has been designed with the main purpose of publishing data about public contracts. The starting point of it has been the Public Contracts Ontology, an ontology developed by the czech Open Data initiative.

The ontology defined in this document can be obtained in RDF/XML format here.

1. Introduction back to ToC

The CONTSEM ontology has been developed under the project "Public procurement optimization through semantic technologies" (CONTSEM as spanish acronym). this project involves the participation of iASoft (OESIA) company, the University of Zaragoza and, as public entities users, the Goverment of Aragón, the Provincial Council of Huesca and the town halls of Huesca and Zaragoza. The main purpose of the project is to add semantic technologies in the software used by public authorities in the procurement procedures. More specifically, one of the core objetives is to describe semantically the information published in official procurement bulletins. For this purpose CONTSEM participants have developed the ontology, focused towards Spain laws and Europe in general.

The ontology reuses various ontologies and schemes to describe some entities, more specifically, we make use of the following ontologies and schemes:

  • Public Contracts Ontology (PCO). Ontology about public procurement developed by the czech Open Data initiative. See also https://code.google.com/p/public-contracts-ontology/.
  • The Organization Ontology. Ontology developed by W3C to describe organizational structures. Actually a W3C Recommendation. See also http://www.w3.org/TR/vocab-org/.
  • Schema.org. Scheme broadly used to describe and extract structured information (e.g. Google, Yahoo). This contains classes and properties to describe a wide set of things. We have used the OWL version of this scheme. See also http://schema.org/.
  • Simple Knowledge Organization System (SKOS). Ontology that provides a model for expressing the basic structure and content of concept schemes (e.g. thesauri, taxonomies). See also http://www.w3.org/2004/02/skos/.
  • Good Relations Ontology. Ontology aimed to describe the various entities of the e-commerce (e.g. objects, prices, companies). See also http://www.heppnetz.de/ontologies/goodrelations/v1.
  • Dublin Core Metadata Terms. Scheme developed by the Dublin Core Metadata Initiative, covers the most basic elements to describe a wide range of resources. See also http://dublincore.org/.

As we can see the contratacion:Contract class may represent both a full contract and a lot of a contract. Surrounding contratacion:Contract class, we may find several classes that are firmly related with the contract itself, i.e. tenderers requirements or the object of the contract. We include this set of classes and properties altogether and name them as Contract box.
